# Proto Workbench Project Rules

以下规则适用于本仓库的原型页面开发与调整，默认优先遵守：

1. 按钮规范
- 按钮使用统一样式。
- 操作按钮默认放在区域右侧。

2. 必填项规范
- 必填字段标签前必须带 `*`。
- `*` 必须使用红色显示。

3. 列表穿透规范（原型）
- 列表以“穿透演示”为主。
- 默认仅第一行实现点击穿透即可。
- 其他行可不实现穿透行为。
- 仅第一行可穿透时，其他行的视觉样式需与第一行保持一致（不做灰色弱化、禁用弱化或样式区分）。
 - 特殊：`/edu/trainer/training-one-person-one-file` 的“一人一档详情”中，“学习详情总览”三条（培训记录/考试记录/证书档案）均可穿透。

4. 原型数据规范
- 列表默认提供约 10 条假数据用于展示。
- 原型页中点击按钮默认不新增真实数据（仅演示交互，不落库）。
- 原型页中的“查询”按钮点击默认不需要产生实际反应（可仅保留按钮展示，不触发数据变化）。

5. 发布中心规范（真实工具）
- 发布中心属于可真实使用的工具页面。
- 需支持基于最新页面实时刷新列表。
- 需支持发布最新设计产物。

6. 文案规范
- 默认不添加括号备注文案（例如“xxx（原型）”“xxx（测试）”）。
- 文案直接、业务化表达，避免额外注释性后缀。

7. 板块字段规范
- 涉及“板块/所属板块/所属版块”的字段，统一使用以下枚举值：
- `石油工程`、`炼油`、`化工`、`销售`、`炼化工程`、`科研`、`其他`
- 原型页中的下拉选项、假数据展示值、默认值需与该枚举保持一致。

8. 时间日期字段规范
- 字段值只到 `YYYY-MM-DD` 的，文案统一使用“日期”命名。
- 字段值包含 `YYYY-MM-DD HH:mm:ss` 的，文案统一使用“时间”命名。
- 原型页面中不得出现“文案写时间、值只有日期”或“文案写日期、值包含时分秒”的混用。

9. 项目维护页面容器与分页规范
- `project-maintenance` 相关页面统一使用白色背板容器（与主页面视觉一致）。
- 凡是列表页都需要展示页码区（分页条），即使是原型假分页也需保留分页展示样式。

10. 页面背板与列表容器规范
- 页面中有业务内容（有文字/表单/列表）的主体区域统一放在白色背板容器内。
- 所有列表区域不显示列表上方标题文案（例如“XXX数据列表”这类字样）。
- 列表区域外层不叠加多层边框容器，列表保持单层边框视觉。

11. 列表分页样式规范（智能判断）
- 主页面列表默认展示分页条，样式采用统一样式（参考：`共X条记录 第Y/Z页` + 上一页/下一页 + 当前页高亮 + `10条/页`下拉）。
- 穿透弹窗（子页面）中的列表默认不展示分页条；仅在业务明确需要时再展示。
- 原型页可使用假分页（不必真实翻页），默认每页条数为 `10条/页`。
- 若需在弹窗中强制展示分页，给列表容器加显式标记（如 `force-pager`），避免自动误加分页。

12. 穿透页字段可编辑态规范（智能判断）
- 穿透页面中，凡是不可编辑字段（`disabled` 或 `readonly`）对应的输入框/下拉框/文本域，统一使用浅灰色背景展示，表示“不可编辑”。
- 不可编辑字段的文字颜色统一使用中灰色，边框使用浅灰色，避免与可编辑字段混淆。
- 该规则适用于所有模块的穿透弹窗与子页面表单。

13. 统一身份字段规范
- 涉及“统一身份/统一身份账号/统一账号”字段时，假数据统一使用“字母+数字”组合。
- 不使用纯中文、不使用纯数字作为统一身份示例值。
- 推荐格式示例：`ZHANGMING320821`、`UIDA2026001`、`LM420602B2`。

14. 查询区排版规范
- 查询条件区域采用紧凑左对齐布局，不保留大块空白。
- `查询`、`重置`按钮默认紧跟最后一个查询条件展示，不与右侧其他业务按钮混排。
- 业务按钮（如新增、导入、导出、模板下载）需与查询按钮分组展示，视觉分区清晰。
- `查询`、`重置`与最后一个查询条件默认同一行展示，不单独另起一行。
- 除`查询`、`重置`外的其他按钮默认右对齐；当一行空间不足时，允许业务按钮组换到下一行并保持右对齐。
- 开发分类页面中，查询类按钮组（查询/重置/收起等）必须紧贴查询条件区域显示，不允许被换到下一行。

15. 开发分类穿透页表格滚动规范
- 教育培训开发分类页面的穿透弹窗内，列表不使用左右滚动条。
- 表格内容通过列宽自适应与单元格换行展示，避免出现横向滑动。

16. 开发分类交互演示规范
- 教育培训开发分类页面中，所有按钮交互默认仅做原型演示，不执行真实落库或持久化写入。
- 列表穿透统一为“仅第一行可穿透”；其余行保持同样视觉样式但不触发穿透行为。
- 点击或悬浮按钮/穿透链接时，不展示下划线、颜色变化、阴影或其他动画反馈。
- 教育培训开发分类页面及其穿透中，所有输入类控件（输入框、下拉框、文本域、单选、复选）无论可编辑或不可编辑，点击/悬浮/聚焦时均不展示颜色变化、阴影、边框高亮或动画反馈。
- 教育培训开发分类页面及其穿透中，不可点击字段（`readonly` / `disabled`）在鼠标悬浮时样式保持完全一致，不允许出现深浅变化、透明度变化、边框高亮、阴影或动画反馈。
- 教育培训开发分类页面及其穿透中，所有按钮（包括`查询`、`重置`、`新增`、`导入`、`导出`、`模板下载`等）在悬浮/点击/聚焦时保持与默认态一致，不允许出现深浅变化、边框变化、阴影、位移或动画反馈。

17. 开发分类日期控件规范
- 教育培训开发分类页面中，日期字段不使用日历组件。
- 日期输入统一使用普通文本输入展示，可保留假数据（例如 `YYYY-MM-DD`）。

18. 总部端与企业端页面职责规范（教育培训）
- 总部端用户定位：集团总部安环主管视角，管理多家下属企业（如 121 家企业）的培训业务。
- 企业端用户定位：单一企业安环主管视角，仅关注本企业培训业务。
- 设计默认差异：
- 总部端页面应保留跨企业维度字段（如所属企业、参与企业、企业筛选等）。
- 企业端页面默认固定“所属企业”为当前企业，不再提供跨企业筛选与跨企业统计字段。
- 企业端查询条件、列表字段、穿透字段需优先保留本企业管理所需信息（如计划名称、培训班、主办/承办部门、计划类型、登记信息），删除仅总部视角才有意义的跨企业字段。
- 总部端与企业端若共用同一业务模板，企业端需先做字段收敛再出图，避免直接复制总部端造成冗余字段。

执行约定：
- 若用户新指令未特别说明，按以上默认规则执行。
- 若用户明确提出与上述规则冲突的新要求，以用户当次明确指令为准。
